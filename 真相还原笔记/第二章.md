# 第二章

* 8086计算机开机时，cpu的cs:ip寄存器被强制转化为0xF000:0xFFF00
* 开机时处于实模式，实模式下的段基址要乘16，也就是左移4位
* 由于cpu访问的内存是：段地址+偏移地址，所以访问的地址是0xF000*16+0xFFF0=0xFFFF0
* mbr（512字节，最后两个字节0x55，0xaa，如果是，则加载扇区内容到0x7c00内存）的任务就是加载loader，由loader加载操作系统到指定位置，然后执行加载过来的操作系统
* 必须是512字节的原因是为了占满512字节

## 代码mbr.S

1. 代码功能

   显示几个字符

2. 实现原理

   用8086汇编语言编写显示字符的程序，共512个字节，不足补0，用nasm编译后，用dd命令写入bochs启动硬盘的0盘0道1扇区

3. 代码逻辑

   * 清屏
   * 获取光标位置
   * 在光标位置处打印字符串

4. 怎么写

   1. 指定起始地址0x7c00——让编译器把本程序的起始地址编译为0x7c00
   2. 调用bios中断清屏
   3. 调用bios中断获取光标位置
   4. 调用bios中断获取打印字符
   5. 死循环，填入mbr规定510字节大小剩下的0，固定结尾两字节0x55，0xaa

## 关于MBR的位置0x7c00等系列疑问

1. 为什么?16位Intel 8086时代,因为当年内存小,为了兼顾内存最小的机器,放在32kb前,计算刚好是0x7c00,0x7c00是为了照顾32kb内存的机器,后面为了兼容,在开机时会伪装为16位的8086

2. 为什么是1mb?因为8086只有20根地址线,所以这就是为什么16位的寄存器,段基址会先向左移动4位到20位,然后再加16位的段内偏移地址

3. 16位的寄存器,拆分为8位寄存器使用,高的叫h,低位叫l

4. intel使用小端存储,位置最低的就是最低的,如果是那种前面有很多0的数据放在一段内存里,可以直接拿过来调用,而不需要再去像大端那样累加的去找

5. 在16位的汇编中,寄存器是分角色的

   1. 通用寄存器(计算和传参)

      ax,bx,cx,dx

      名字以x结尾,可以拆分为高低8位

   2. 段寄存器(定位内存地址)

      cs,ds,es,ss,fs,gs(代码段寄存器,数据段寄存器,附加段寄存器,ss:堆栈段寄存器,分别记录其段基址)

      名字以s结尾

6. 区分,对于汇编来说
   1. 段寄存器是地图,决定了访问时去哪里找
   2. 通用寄存器是快递盒,是用来给BIOS中断塞参数的

7. bp全称是Base Pointer(基址指针寄存器)->专门用来指向内存地址

8. 使用int 0x10的0x13功能(显示字符串时),规定,只去es:bp组合起来的地址找

9. 汇编里面存在定义字节的伪指令

   | **伪指令** | **全称**               | **占用空间**      | **适用场景**           |
   | ---------- | ---------------------- | ----------------- | ---------------------- |
   | **`db`**   | Define **Byte**        | **1 字节** (8位)  | 字符、字符串、小数字   |
   | **`dw`**   | Define **Word**        | **2 字节** (16位) | 16位整数、内存偏移地址 |
   | **`dd`**   | Define **Double word** | **4 字节** (32位) | 32位整数、浮点数       |
   | **`dq`**   | Define **Quad word**   | **8 字节** (64位) | 64位长整数             |

   db是给NASM看的,告诉编译器,放原始数据

10. 为什么1字节8位?

    1964年IBM大型机的行业标准,使用了最少的位数完整支持了ASCII字符集(7位,高位补0),并且兼顾了2的幂次方
