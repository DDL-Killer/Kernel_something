## 什么是地址?

* 地址只是数字,描述各种符号在源程序中的位置,它是源代码文件中各个符号偏移开头
* 编译器的工作就是给各个符号编址,编译器根据所在硬件平台的特性,将源码中的符号(指令和数据)按照平台的特性分配空间
* 程序中各种数据结构的访问,通过"该数据结构的起始地址+该数据结构所占的内存大小"实现----这就是为什么定义变量要给出变量类型,因为变量类型规定了变量所占的内存大小
* 占据在第一位的数据,便是整个程序的起始地址
* 数据的地址,其实就是该数据相对整个程序开头的距离,即偏移量
* 偏移量的本质就是地址,每个变量的地址是前一个变量的地址+前一个变量的内存空间大小
* 编译器给程序中各符号分配的地址,就是各符号相对于文件开头的偏移量

## 什么是section?

* 编译器提供的关键字只是为了让程序员在逻辑上将程序划分为几个区域,是伪指令
* 关键字section并没有对程序中的地址产生任何影响,即在默认情况下,也没有section都一样,section中数据的地址仍然是相对于整个文件的顺延,仅仅是在逻辑上让开发人员梳理程序之用

## 什么是vstart

* vstart是虚拟起始地址
* 是为section内的数据指定一个虚拟的起始地址
* 编译器以相对于文件开头偏移来编址的好处是利于重定位,整个文件加载到某个地址后,可以以这个地址为段基址
* 程序只有加载到非0地址时vstart才是有用的,程序默认起始地址是0
* vstart只是告诉编译器以新的数字作为后面数据的地址的起始值,它本身没办法改变数据本身在文件中的地址

## CPU的实模式

### CPU的工作原理

大体上分为三个部分,控制单元,运算单元,存储单元

* 控制单元是CPU的控制中心,由指令寄存器IR,指令译码器ID,操作控制器OC组成,程序加载到内存后,指令指针寄存器IP指向内存下一条待执行指令的地址,控制单元根据IP寄存器的指向,将位于内存中的指令逐个装载到指令寄存器中,指令译码器将位于指令寄存器中的指令按照指令格式来解码

  指令格式

  | 前缀 | 操作码 | 寻址方式 操作数类型 | 立即数 | 偏移量 |
  | ---- | ------ | ------------------- | ------ | ------ |
  |      |        |                     |        |        |

* 存储单元是指CPU内部的L1 L2缓存及寄存器,待处理的数据就存放在这些存储单元里面
  * 缓存采用的都是SRAM(static RAM),主板上属于DRAM(Dynamic Random Access Memory)
  * 寄存器分为两大类
    * 可见寄存器
    * 不可见寄存器
* 过程
  * 用户进程的代码段映射到内存,内存上有很多指令,CPU的控制单元中的操作控制器OC从内存中取得命令后,由指令指针寄存器IP得到待运行的指令地址
  * 交给指令寄存器,指令寄存器得到指令开始解码后发给运算单元
  * 存储单元从内存中获得操作数
* 总结:控制单元取下一条待运行的指令,该指令在程序计数器PC中,cs:ip,读取ip后,将地址送上地址总线,CPU得到了指令,确定操作码,再确定操作数类型,如果在内存中,将操作数取回放到自己的存储单元,操作控制器给运算单元下令执行,ip寄存器继续加上当前指令大小恢复先前操作继续循环

### 实模式下的寄存器

* 寄存器是一种物理存储元件,比一般的存储介质要快
* 缓存解决了速度不匹配设备之间的数据传输
* 浏览器在请求一个域名IP时
  * 浏览器内部有dns客户端,查询本地dns缓存中是否有该域名的ip,有就直接访问,没有再去查找自己主机所设置的dns服务器,然后去该dns服务器查询
  * 如果dns服务器本地缓存有该域名的A记录,直接返回浏览器dns,没有就递归向上查询
  * 浏览器的dns客户端在得到ip后,将该域名和ip放到自己的缓存
  * 浏览器开始通过网络用http协议访问该ip地址的80端口
  * ip地址对应的设备不是最终的Web服务器,一般是网关,由网关检查本地缓存中是否有相关Web服务器的缓存,如果有就请求分配给缓存中的Web服务器,随后将web服务器和ip地址和端口号缓存
  * web服务器拿到请求,静态就检查,直接取,动态就交给自己的cgi处理
  * cgi先看自己的缓存,没有的话,与数据库建立连接,发出请求
  * 数据库先检查自己的缓存,没有的话从结果中检索返回,将结果集缓存
  * cgi拿到数据,返回Web服务器,存缓存
  * Web服务器将数据返回网关
  * 网关返回浏览器
  * 浏览器发现有静态数据,存到internet临时目录
* 如果缓存刚好有需要的内容,称其为命中hit,否则就是缺失mis
* 寄存器分两类
  * 内部使用,不可见,全局描述符表寄存器GDTR 中断描述符寄存器IDTR 局部描述符表寄存器LDTR 任务寄存器CR0~3 指令指针寄存器IP 标志寄存器flags 调试积存器DR0~7
  * 可见的寄存器,段寄存器 通用寄存器
