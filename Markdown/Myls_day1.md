## `getopt()`函数

* `getopt()`函数是标准库`<unistd.h>`提供的专门用来标准化解析命令行参数的函数

  ```c
  #include <unistd.h>
  
  int getopt(int argc, char * const argv[], const char *optstring);
  /*
  关键参数:optstring
  单个参数:表示这是一个开关,不需要参数 &quot;a&quot;表示接受-a
  字符后跟一个冒号:表示选项后面必须跟一个参数值
  字符后跟两个冒号:表示参数是可选的
  */
  ```

| 变量名 | 类型     | 作用                                      |
| ------ | -------- | ----------------------------------------- |
| optarg | `char *` | 指向当前选项的*参数值*                    |
| optind | `int`    | 下一个将被处理的argv元素的下标            |
| optopt | `int`    | 当发现未知选项时,存储那个位置的字符       |
| opterr | `int`    | 设为0可禁止getopt向stderr输出默认错误信息 |

## `dirent`库

```c
struct dirent {
    ino_t          d_ino;       /* Inode 编号 (文件系统的索引节点) */
    off_t          d_off;       /* 目录文件开头的偏移量 (不代表文件大小) */
    unsigned short d_reclen;    /* 当前 dirent 记录的长度 */
    unsigned char  d_type;      /* 文件类型 (并非所有文件系统都支持) */
    char           d_name[256]; /* 文件名 (以空字符 \0 结尾) */
};
```

| **成员变量**   | **类型**         | **作用与含义**                                               |
| -------------- | ---------------- | ------------------------------------------------------------ |
| **`d_ino`**    | `ino_t`          | **Inode 编号**。文件在磁盘上的唯一标识。`ls -i` 命令显示的就是这个值。 |
| **`d_off`**    | `off_t`          | **目录偏移量**。指向目录流中下一个 `dirent` 的位置，通常由系统底层使用（如 `telldir`）。 |
| **`d_reclen`** | `unsigned short` | **记录长度**。由于文件名长度不同，每个 `dirent` 结构体在内存中占用的实际大小可能不同。 |
| **`d_type`**   | `unsigned char`  | **文件类型**。这是个“神技”，可以让你不用调用 `stat` 就能判断是目录还是文件（见下文）。 |
| **`d_name`**   | `char[]`         | **文件名**。这是最重要的成员，存储了该条目的名称。           |

## `-a`实现思路

**核心逻辑：过滤隐藏文件** 在 `readdir` 的循环中，通过判断文件名的首字符和 `flags` 的状态来决定是否跳过打印。

- **逻辑规则**：
  - 如果文件名第一个字符是 `.` **且** 用户**没有**输入 `-a` 选项，则跳过（continue）。
  - 否则，正常打印。
