# ls收尾

## 两个重要的函数

### `stat()`

```c
#include<sys/types.h>
#include<sys/stat.h>
#inlude<unistd.h>

int stat(const char *pathname,struct stat *statbuf);
```

* pathname:文件的路径(字符串)
* statbuf:指向`struct stat`结构体的指针,函数会将获取到的信息填入这个结构体中
* 返回值:成功返回0,失败返回-1并设置`errno`

* `stat()`会穿透,如果`pathname`是一个符号链接,它会跟踪(follow)这个链接,返回的是指向目标文件的信息
* 用途:想要读取文件的实际内容或属性时使用

## `lstat()`

```c
#include<sys/types.h>
#include<sys/stat.h>
#inlude<unistd.h>

int stat(const char *pathname,struct stat *statbuf);
```

* 如果 `pathname` 是一个符号链接，`lstat()` **不会追踪**链接，它返回的是**符号链接文件本身**的信息。

* *用途：* 当你需要查看链接本身的信息（例如链接本身占用的空间、链接的权限）或者在编写像 `ls -l`、`tar` 备份工具时使用（防止陷入循环链接或备份了重复数据）

## `struct stat`结构体

```c
struct stat {
    dev_t     st_dev;     // 包含该文件的设备 ID
    ino_t     st_ino;     // Inode 节点号 (文件的唯一标识)
    mode_t    st_mode;    // 文件的类型和权限 (非常重要!)
    nlink_t   st_nlink;   // 硬链接数量
    uid_t     st_uid;     // 所有者的用户 ID
    gid_t     st_gid;     // 所有者的组 ID
    dev_t     st_rdev;    // 设备 ID (如果是特殊设备文件)
    off_t     st_size;    // 文件大小 (字节)
    blksize_t st_blksize; // 文件系统 I/O 的块大小
    blkcnt_t  st_blocks;  // 分配的 512B 块数量
    
    // 时间戳 (在较新的 Linux 内核中精度为纳秒)
    struct timespec st_atim;  // 最后访问时间 (Access Time)
    struct timespec st_mtim;  // 最后修改时间 (Modification Time)
    struct timespec st_ctim;  // 状态改变时间 (Change Time)
};
```

无论调用哪个函数,成功后都会填充此结构体

## stat结构中的st_mode参数

| 宏         | 文件类型     |
| ---------- | ------------ |
| S_ISREG()  | 普通文件     |
| S_ISDIR()  | 目录文件     |
| S_ISCHR()  | 字符特殊文件 |
| S_ISBLK()  | 块特殊文件   |
| S_ISFIFO() | 管道或FIFO   |
| S_ISLNK()  | 符号链接     |
| S_LSSOCK() | 嵌套字       |

## 2. 查表翻译：解密 `uid` 与 `gid`

在 `stat` 结构体中，文件的所有者和所属组被记录为 `st_uid` 和 `st_gid`，它们在底层只是毫无感情的整型数字。为了把“身份证号”还原成“真实姓名”，我学习了 Linux 系统编程中的用户和组信息查询 API。

我们需要依赖 `<pwd.h>` 和 `<grp.h>` 这两个头文件，并通过下面两个核心系统调用函数，去系统的“底层户籍库”里查表：

## 用户信息查表：`getpwuid()`

- **传入参数**：`uid_t uid` (比如 `1000`)
- **返回值**：返回一个指向 `struct passwd` 结构体的指针。如果查无此人，返回 `NULL`。

**`struct passwd` 核心参数解析库：**

| 成员变量      | 数据类型 | 资料库含义                             | 在 ls -l 中的用途                  |
| ------------- | -------- | -------------------------------------- | ---------------------------------- |
| **`pw_name`** | `char *` | **用户的登录名** (如 `root`, `ubuntu`) | **【核心提取目标】打印文件所有者** |
| `pw_passwd`   | `char *` | 加密后的密码 (通常用 `x` 占位)         | 本次不用                           |
| `pw_uid`      | `uid_t`  | 用户的 UID 数字                        | 本次不用                           |
| `pw_gid`      | `gid_t`  | 用户所在主组的 GID 数字                | 本次不用                           |
| `pw_dir`      | `char *` | 用户的主目录 (如 `/home/ubuntu`)       | 本次不用                           |
| `pw_shell`    | `char *` | 用户的默认 Shell (如 `/bin/bash`)      | 本次不用                           |

### 组信息查表：`getgrgid()`

- **传入参数**：`gid_t gid`
- **返回值**：返回一个指向 `struct group` 结构体的指针。如果查无此组，返回 `NULL`。

**`struct group` 核心参数解析库：**

| 成员变量      | 数据类型 | 资料库含义                        | 在 ls -l 中的用途                  |
| ------------- | -------- | --------------------------------- | ---------------------------------- |
| **`gr_name`** | `char *` | **组的名称** (如 `root`, `staff`) | **【核心提取目标】打印文件所属组** |
| `gr_passwd`   | `char *` | 组密码                            | 本次不用                           |
| `gr_gid`      | `gid_t`  | 组的 GID 数字                     | 本次不用                           |

导出到 Google 表格

**我的代码实现避坑：** 因为这两个函数有可能返回空指针（比如某个文件所属的用户被系统删除了），直接访问 `pw->pw_name` 会导致段错误。所以我使用了三目运算符 `pw ? pw->pw_name : "unknown"` 进行安全兜底。

## 3. 时间的“变形”：`localtime` 与 `strftime` 占位符艺术

`st_mtime` 拿出来是一长串数字（从 1970 年 1 月 1 日算起的秒数时间戳）。为了把秒数变成我们熟悉的格式，我使用了一套“组合拳”：

**第一步：解构时间戳 `localtime()`** 将 `time_t` 类型的秒数，传入 `localtime(&st_mtime)`，它会返回一个拆解好的 `struct tm` 结构体，里面把年、月、日、时、分、秒都分门别类地算好了。

**第二步：格式化缝合 `strftime()`** 拿到 `struct tm` 后，需要使用 `strftime()` 函数将时间拼成字符串。这个函数就像是给时间定制衣服的裁缝，通过传入特定的一系列 `%` 占位符，来精准控制输出格式。

为了复刻真正的 `ls -l` 输出效果，我使用了 `"%b %e %H:%M"` 这个格式串。以下是我整理的时间格式化占位符资料库：

**`strftime` 常用占位符资料库：**

| 占位符   | 代表的含义与格式                      | 实际输出举例                  | 为什么 ls 采用它？                                           |
| -------- | ------------------------------------- | ----------------------------- | ------------------------------------------------------------ |
| **`%b`** | **月份的本地化简写**                  | `Oct` (英文) 或 `10月` (中文) | 这是 `ls -l` 标准的月份展示方式。                            |
| **`%e`** | **月份中的第几天 (1-31)，带空格补齐** | ` 5` 或 `24`                  | 坑点：如果是 5 号，前面会补一个**空格**而不是 0。这样可以保证所有日期列绝对右对齐。 |
| `%d`     | 月份中的第几天 (01-31)，带0补齐       | `05` 或 `24`                  | `ls` 不用它，因为 `05` 看起来不够纯粹。                      |
| **`%H`** | **24 小时制的小时 (00-23)**           | `14`                          | 国际通用的时间显示标准。                                     |
| **`%M`** | **分钟 (00-59)**                      | `30`                          | 配合小时，组成 `14:30` 的显示效果。                          |
