# 目标

多线程I/O与并发控制,构建并发文件搜索工具,调用底层API,在多线程环境下保护共享资源,优雅的控制并发数

## <semaphore.h>

POSIX标准定义的信号量头文件,主要用在多线程或多进程编程中,解决同步与互斥问题

### 核心操作

信号量的使用,通常遵循"创建初始化"->"P操作与V操作"->"销毁"

| 步骤       | 操作 (函数)     | 功能描述                                                     | 比喻                                                         |
| ---------- | --------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **初始化** | `sem_init()`    | 初始化一个**无名信号量**，并设定初始计数值（可用资源总数）   | 停车场门口立个牌子，写上总车位数（初始值）                   |
| **P操作**  | `sem_wait()`    | **等待** 信号量值减1 若减后为负数，线程将被阻塞，直到资源可用 | 一辆车进入，车位剩余数减1 如果没有车位了，车辆就在门口排队等待 |
| **V操作**  | `sem_post()`    | **释放** 信号量值加1 若有其他线程在等待，则唤醒其中一个      | 一辆车离开，车位剩余数加1，并通知排队的一辆车进入            |
| **销毁**   | `sem_destroy()` | 销毁初始化阶段创建的无名信号量，释放相关资源                 | 停车场被拆除，牌子也被扔掉                                   |

> `sem_trywait()` 和 `sem_timedwait()` 是 `sem_wait()` 的变种。`trywait` 尝试获取，失败则立即返回，不会阻塞；`timedwait` 则只等待指定的时间

### API速查表

| 函数              | 适用类型       | 主要功能                                            |
| ----------------- | -------------- | --------------------------------------------------- |
| `sem_init()`      | **无名信号量** | 初始化一个无名信号量                                |
| `sem_destroy()`   | **无名信号量** | 销毁一个无名信号量                                  |
| `sem_open()`      | **命名信号量** | 创建或打开一个命名信号量，通常用于不相关进程间同步  |
| `sem_close()`     | **命名信号量** | 关闭一个由 `sem_open()` 打开的命名信号量            |
| `sem_unlink()`    | **命名信号量** | 从系统中移除一个命名信号量                          |
| `sem_wait()`      | 通用           | **P操作** 等待，信号量值减1，资源不可用时阻塞       |
| `sem_trywait()`   | 通用           | `sem_wait()` 的非阻塞版本，资源不可用时立即返回错误 |
| `sem_timedwait()` | 通用           | `sem_wait()` 的限时版本，超过指定时间即返回         |
| `sem_post()`      | 通用           | **V操作** 释放，信号量值加1并唤醒等待线程           |
| `sem_getvalue()`  | 通用           | 获取当前信号量的值（通常用于调试）                  |

### 无名 VS 命名信号量

| 特性         | 无名信号量 (Unnamed Semaphore)                         | 命名信号量 (Named Semaphore)                                 |
| ------------ | ------------------------------------------------------ | ------------------------------------------------------------ |
| **适用场景** | 单个进程内的**多线程同步**，或通过共享内存的进程间同步 | **不相关进程**之间的同步                                     |
| **标识方式** | 内存中的变量 (如 `sem_t sem`)                          | 一个以 `/` 开头的**名字** (如 `/my_sem`)                     |
| **生命周期** | 由程序员控制，随进程或共享内存区域的存在而存在         | 由内核管理，即使所有进程关闭它，也需显式调用 `sem_unlink()` 删除 |
| **主要函数** | `sem_init()`, `sem_destroy()`                          | `sem_open()`, `sem_close()`, `sem_unlink()`                  |

### 编译时加上 `-pthread`

## strncmp()

字符串比较函数,可以指定比较的字符数

```c
#include <string.h>
int strncmp(const char *s1, const char *s2, size_t n);
```

#### 参数说明

* s1:第一个字符串
* s2:第二个字符串
* n:最多比较的字符数

## opendir()

打开目录的函数,返回一个目录流指针,用于后续读取目录内容

```c
#include<dirent.h>
DIR *opendir(const char *name)
```

参数:name - 目录路径

返回值:

​	成功:返回`DIR*`类型的目录流指针

​	失败:返回`NULL`,并设置`error`

## readdir()

用于读取目录流的下一个目录项,是遍历目录的核心函数

```c
#include<dirent.h>
struct dirent *readdir(DIR *dirp);
```

参数:由`opendir()`返回的目录流指针

返回值:

​	成功:返回指向`struct dirent`的指针(指向内部静态数据)

​	读到目录末尾:返回`NULL`(不会设置errno)

​	出错:返回`NULL`(会设置error)

## snprintf()

安全格式化字符串函数,可以限制输出长度,防止缓冲区溢出

```c
#include<stdio.h>
int snprintf(char *str,size_t size,const char *format,...);
```

#### 参数说明

* `str`:目标缓冲区指针
* `size`:缓冲区的大小,包括结尾的`\0`
* `format`:格式化字符串(与printf相同)
* `...`:可变参数

#### 返回值

* 成功:返回实际需要的字符数(不包括结尾的`\0`)
* 失败:返回负数(通常是`-1`)

## 线程函数原理

```c
遇到子目录
    ↓
创建新任务
    ↓
【判断】当前线程数 < 最大线程数 且 获取信号量成功？
    ↓                           ↓
   YES                         NO
    ↓                           ↓
创建新线程处理        【降级】当前线程直接处理
    ↓                (深度优先，同步遍历)
新线程并发执行
```

### 关键概念解析

1. 并发令牌机制

   ```c
   sem_t thread_sem;  // 信号量，控制最大并发数
   // 假设 MAX_THREADS = 5，初始值 = 5
   // sem_trywait() 尝试获取一个令牌
   // - 成功：信号量值减1，允许新线程
   // - 失败：信号量值为0，说明已有5个线程在运行
   ```

2. 两种执行模式

   | 模式         | 条件       | 行为             | 特点                   |
   | ------------ | ---------- | ---------------- | ---------------------- |
   | **并发模式** | 有空闲令牌 | 创建新线程       | 并行处理，效率高       |
   | **降级模式** | 无空闲令牌 | 当前线程直接处理 | 串行处理，防止线程爆炸 |

## sem_trywait()

信号量的非阻塞等待函数,尝试获取信号量,如果无法立即获取即立即返回错误

```c
#include<semaphore.h>
int sem_trywait(sem_t *sem);
```

#### 参数和返回值

* 参数:`sem`-信号量指针
* 返回值:
  * 0:成功获取,信号量-1
  * -1:失败,设置errno

#### 与sem_wait()的对比

| 函数              | 行为                       | 适用场景                     |
| ----------------- | -------------------------- | ---------------------------- |
| `sem_wait()`      | 阻塞等待，直到能获取信号量 | 必须获取资源才能继续         |
| `sem_trywait()`   | 立即返回，不阻塞           | 能获取就获取，不能就做其他事 |
| `sem_timedwait()` | 等待指定时间               | 有超时限制的场景             |

## sem_post()

信号量的释放函数,将信号量的值加1,并唤醒等待该信号量的线程

```c
#include <semaphore.h>
int sem_post(sem_t *sem);
```

#### 参数和返回值

* 参数:`sem`-信号量指针
* 返回值:
  * 0:成功
  * -1:失败,设置errno

## pthread_mutex_lock()

获取互斥锁的函数,如果锁已经被其他线程占用,则阻塞等待直到锁可用

```c
#include <pthread.h>
int pthread_mutex_lock(pthread_mutex_t *mutex);
```

* mutex:互斥锁指针

## pthread_mutex_unlock()

释放互斥锁的函数,使其他线程可以获得该锁

```c
#include<pthread.h>
int pthread_mutex_unlock(pthread_mutex_t *mutex);
```

## sem_init()

初始化一个未命名的信号量,这是调用信号量之前必须要调用的函数

```c
#include<semaphore.h>
int sem_init(sem_t *sem, int pshared,unsigned int value);
```

#### 参数说明

| 参数      | 说明                                   |
| --------- | -------------------------------------- |
| `sem`     | 指向要初始化的信号量变量               |
| `pshared` | 共享标志：0=线程间共享，非0=进程间共享 |
| `value`   | 信号量的初始值（0 到 SEM_VALUE_MAX）   |

## pthread_mutex_init()

动态初始化互斥锁,在使用互斥锁之前必须调用此函数

```c
#include <pthread.h>
int pthread_mutex_init(pthread_mutex_t *mutex, const pthread_mutexattr_t *attr);
```

#### 参数说明

| 参数    | 说明                            |
| ------- | ------------------------------- |
| `mutex` | 指向要初始化的互斥锁变量        |
| `attr`  | 互斥锁属性（NULL 表示默认属性） |

### 互斥锁类型对比

| 类型                       | 特性                 | 使用场景   |
| -------------------------- | -------------------- | ---------- |
| `PTHREAD_MUTEX_NORMAL`     | 默认，不检查错误     | 一般用途   |
| `PTHREAD_MUTEX_RECURSIVE`  | 允许同一线程重复加锁 | 递归函数   |
| `PTHREAD_MUTEX_ERRORCHECK` | 提供错误检查         | 调试阶段   |
| `PTHREAD_MUTEX_DEFAULT`    | 实现定义的行为       | 可移植代码 |

## sem_destroy()

销毁未命名的信号量,释放信号量占用的资源

```c
#include <semaphore.h>
int sem_destroy(sem_t *sem);
```

## pthread_mutex_destory()

销毁互斥锁,释放互斥锁占用的资源

```c
#include <pthread.h>
int pthread_mutex_destroy(pthread_mutex_t *mutex);
```
