## 终端中ANSI转义序列输出控制文本颜色和样式

### 完整格式说明

```c
\033[ 样式1;样式2;...;前景色;背景色 m

\033[0m     重置
\033[1;4m   粗体 + 下划线
\033[31m    红色文字
\033[31;42m 红色文字 + 绿色背景
```

### 样式控制码

| 代码 | 效果                     |
| ---- | ------------------------ |
| `0`  | 重置所有样式             |
| `1`  | 粗体/高亮                |

| `6`  | 快速闪烁（不常用）       |
| `7`  | 反色（交换前景/背景色）  |
| `8`  | 隐藏（不可见）           |
| `9`  | 删除线代码               |

## &

* 在shell中,`&`的作用是将命令放在后台运行
* 加上shell中,shell会立即启动这个命令,但不会等它结束,而是立即返回提示符

注意事项:

	1. 后台任务仍然会向终端输出,可能打断输入
	1. 退出终端后,后台任务可能被终止
	3. `&`的优先级一般非常低,写在命令末尾



在main()函数中的执行命令前,通过辨别命令末尾的`&`,将其删除设置为NULL,并且增加一个`is_background = 1`来应对有`&`的情况



## dup2()

```c
#include<unistd.h>
int dup2(int oldfd,int newfd);
```

将`newfd`重定向,使其指向与`oldfd`相同的文件表项(复制文件描述符)

* 让newfd成为oldfd的副本
* 如果newfd已经打开,会先关闭
* 两个描述符共享文件偏移量和文件状态标志
* 成功返回newfd

## 重定向

### 标准文件描述符

| 文件描述符 | 名称              | 符号     | 默认指向 |
| ---------- | ----------------- | -------- | -------- |
| 0          | 标准输入 (stdin)  | `<`      | 键盘     |
| 1          | 标准输出 (stdout) | `>` `>>` | 终端屏幕 |
| 2          | 标准错误 (stderr) | `2>`     | 终端屏幕 |

### 操作符总结

| 操作符 | 说明                    | 示例                    |
| ------ | ----------------------- | ----------------------- |
| `>`    | 重定向 stdout（覆盖）   | `ls > list.txt`         |
| `>>`   | 重定向 stdout（追加）   | `echo "log" >> log.txt` |
| `2>`   | 重定向 stderr（覆盖）   | `err 2> error.txt`      |
| `2>>`  | 重定向 stderr（追加）   | `err 2>> error.txt`     |
| `&>`   | 重定向 stdout 和 stderr | `cmd &> all.txt`        |
| `>&`   | 复制文件描述符          | `2>&1`                  |
| `<`    | 输入重定向              | `sort < data.txt`       |
| `<<`   | Here Document           | `cat << EOF`            |
| `<<<`  | Here String             | `bc <<< "2+3"`          |
| `|`    | 管道                    | `cmd1 | cmd2`           |
| `|&`   | 管道（合并 stderr）     | `cmd1 |& cmd2`          |

## 权限计算

Linux/Unix中文件的权限使用八进制数字

### 每个权限对应一个二进制位

| 权限  | 二进制 | 八进制 | 含义       |
| ----- | ------ | ------ | ---------- |
| `---` | 000    | 0      | 无任何权限 |
| `--x` | 001    | 1      | 仅执行     |
| `-w-` | 010    | 2      | 仅写入     |
| `-wx` | 011    | 3      | 写入+执行  |
| `r--` | 100    | 4      | 仅读取     |
| `r-x` | 101    | 5      | 读取+执行  |
| `rw-` | 110    | 6      | 读取+写入  |
| `rwx` | 111    | 7      | 所有权限   |

### 权限位分配

```c
0644 = 二进制: 110 100 100
              │   │   │
              │   │   └─ 其他用户 (others) : 100 = 4 = r--
              │   └───── 组用户 (group)    : 100 = 4 = r--
              └───────── 所有者 (owner)    : 110 = 6 = rw-
```

### 常见权限对照表

| 八进制 | 二进制      | 符号表示    | 说明                     |
| ------ | ----------- | ----------- | ------------------------ |
| 000    | 000 000 000 | `---------` | 无任何权限               |
| 400    | 100 000 000 | `r--------` | 仅所有者可读             |
| 600    | 110 000 000 | `rw-------` | 所有者可读写             |
| 644    | 110 100 100 | `rw-r--r--` | 所有者读写，其他人只读   |
| 644    | 110 100 100 | `rw-r--r--` | **最常用的文件权限**     |
| 640    | 110 100 000 | `rw-r-----` | 所有者读写，组只读       |
| 700    | 111 000 000 | `rwx------` | 仅所有者全权             |
| 750    | 111 101 000 | `rwxr-x---` | 所有者全权，组可读执行   |
| 755    | 111 101 101 | `rwxr-xr-x` | 所有者全权，其他人读执行 |
| 777    | 111 111 111 | `rwxrwxrwx` | 所有人全权（不安全）     |
| 600    | 110 000 000 | `rw-------` | 私密文件                 |
| 666    | 110 110 110 | `rw-rw-rw-` | 日志文件                 |

### 特殊权限位

```c
// 完整的权限是 4 位八进制
0755  // 前导0表示没有特殊权限
4755  // 4 = setuid 权限
2755  // 2 = setgid 权限
1755  // 1 = sticky bit (粘滞位)

// 特殊权限位
setuid (4): 执行时以文件所有者身份运行
setgid (2): 执行时以文件组身份运行
sticky (1): 只有文件所有者能删除（用于/tmp目录）
```

## open()

```c
#include <fcntl.h>
#include <sys/stat.h>
#include <unistd.h>

// 打开已有文件
int open(const char *pathname, int flags);

// 创建新文件（需要指定权限）
int open(const char *pathname, int flags, mode_t mode);
```

打开或创建文件,返回文件描述符

### 全部的flags

```c
#include<fcntl.h>
```

#### **访问模式标志（必须三选一）**

| 标志       | 值   | 说明     |
| ---------- | ---- | -------- |
| `O_RDONLY` | 00   | 只读打开 |
| `O_WRONLY` | 01   | 只写打开 |
| `O_RDWR`   | 02   | 读写打开 |

#### 文件创建标志

| 标志       | 值    | 说明                                  |
| ---------- | ----- | ------------------------------------- |
| `O_CREAT`  | 0100  | 文件不存在则创建                      |
| `O_EXCL`   | 0200  | 与 O_CREAT 一起使用，文件已存在则失败 |
| `O_NOCTTY` | 0400  | 如果路径是终端设备，不分配控制终端    |
| `O_TRUNC`  | 01000 | 如果文件存在且以写方式打开，清空内容  |

#### 文件状态标志

| 标志          | 值        | 说明                           |
| ------------- | --------- | ------------------------------ |
| `O_APPEND`    | 02000     | 追加模式，每次写入都追加到末尾 |
| `O_NONBLOCK`  | 04000     | 非阻塞模式（POSIX）            |
| `O_NDELAY`    | 04000     | 同 O_NONBLOCK（System V）      |
| `O_SYNC`      | 010000    | 同步写入（等待数据和元数据）   |
| `O_DSYNC`     | 010000    | 同步写入（只等待数据，POSIX）  |
| `O_RSYNC`     | 010000    | 同步读取（POSIX）              |
| `O_ASYNC`     | 020000    | 启用信号驱动 I/O（SIGIO）      |
| `O_DIRECT`    | 040000    | 直接 I/O，绕过页缓存           |
| `O_LARGEFILE` | 0100000   | 支持大文件（>2GB）             |
| `O_DIRECTORY` | 0200000   | 如果路径不是目录则失败         |
| `O_NOFOLLOW`  | 0400000   | 不跟踪符号链接                 |
| `O_CLOEXEC`   | 02000000  | 执行 exec 时自动关闭           |
| `O_PATH`      | 010000000 | 仅获取文件路径，不打开文件     |
| `O_TMPFILE`   | 020000000 | 创建临时文件（Linux 3.11+）    |

#### Linux特有标志

| 标志            | 值        | 说明                     |
| --------------- | --------- | ------------------------ |
| `O_NOATIME`     | 01000000  | 读取文件时不更新访问时间 |
| `O_NOLINK`      | 04000000  | 不允许硬链接（未实现）   |
| `O_LOOP_NOSIZE` | 010000000 | 循环设备特殊标志         |
